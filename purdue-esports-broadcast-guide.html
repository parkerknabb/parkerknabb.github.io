System Overview

The Purdue Esports broadcast system is split across two spaces: the production studio (permanent) and the casting set (multi-purpose room, set up and torn down each show). This guide covers the casting set and its integration with the studio.

Troubleshooting

Click any issue to expand. Work through steps in order.

Camera ▼
Key indicator: If the Elgato "No Input" message does not appear on a camera source, the Thunderbolt box is not communicating with the laptop — it's not an OBS input issue.
  1. Check that the Sonnet Breakaway Box has power (indicator light on).
  2. Reseat the black Thunderbolt USB-C cable — ensure fully seated on both ends, using a rear USB-C port on the laptop.
  3. Shut down the casting laptop completely, then restart it with the Thunderbolt box already connected and powered.
  4. After reboot, open Elgato Capture Utility. If camera feeds appear there, the hardware connection is good — the problem is OBS input config. Proceed to step 5.
  5. In OBS, for each camera source: right-click → Properties → verify Device is set to CAM LINK HDMI 1 / CAM LINK HDMI 2 / CAM LINK HDMI 3 respectively.
  6. If Elgato Capture Utility also shows no feeds, the Thunderbolt connection is still not established — repeat steps 1–3, checking the cable for damage.
🌐 Camera / OBS feed is laggy or dropping frames
Network ▼

This is almost always a network throughput issue — the OMT stream is bandwidth-heavy.

  1. Check the current ethernet connection method (adapter vs. direct cable).
  2. If using the USB-Ethernet adapter, try switching to a direct ethernet connection, or vice versa.
  3. If still laggy, physically run a cable from the casting set to the switch under OBS 1 in the studio for a more reliable path.
  4. Confirm no other devices are saturating the network segment during the show.
📺 Casting TVs not showing stream output
Display ▼
  1. In OBS, right-click the OMT Source → Open Source Projector.
  2. In the display selector, choose the TV monitor (not the laptop display).
  3. If you accidentally selected the laptop display, press ESC immediately to undo without affecting the TVs.
  4. Verify the HDMI cable from the laptop is going into the input of the HDMI switcher (not an output port).
  5. Confirm the HDMI switcher is powered on and outputting to both TVs.
🎙 Audio not reaching vMix / caster levels missing
Audio ▼
  1. Confirm the RODECaster Pro II was powered on before vMix was opened. If vMix launched first, close vMix, power cycle the RODECaster, then reopen vMix.
  2. In vMix, verify the audio input device is set to the RODECaster USB interface.
  3. Check XLR mic cables and ¼" return cables running through the loom — trace for any disconnected ends at the headset or board end.
  4. On the RODECaster, confirm the caster channel faders are up and not muted.
  5. If the RODECaster USB disconnects mid-show: unplug and replug the USB cable, then re-select the device in vMix audio settings.
  6. Check that no channels are accidentally solo'd on the board — a solo'd channel will prevent you from hearing other sources in your producer headset.
🎧 Caster only hearing audio in one ear
Audio ▼

This is almost always the ¼" headset return cable being slightly unseated — a known quirk of this connector type.

  1. Gently twist the ¼" cable at the headset connection point while the caster listens — rotate slowly until stereo is restored.
  2. If twisting doesn't resolve it, fully unplug and firmly reseat the ¼" connector.
  3. If the issue persists on one specific caster, try swapping their ¼" cable with the spare if available, to rule out a damaged cable.
🔇 Can't hear something you should in producer headset
Audio ▼
  1. Check the RODECaster for any solo'd channels — when a channel is soloed, you only hear that channel. Unsolo everything and check again.
  2. Confirm the channel for the source you're not hearing (casters, game audio, music) has its fader up and is not muted.
  3. Verify game audio from Observer PCs is routing through vMix and being sent to the RODECaster mix.
  4. Remember: the producer headset (3rd fader) is configured to never go to stream — if something is not on stream but you can hear it, that's expected behavior.
🖥 vMix not receiving OMT from casting set
Network ▼
  1. Confirm OBS is running on the casting laptop and the OMT output is active (not just OBS being open).
  2. Verify the casting laptop's network connection is live — check for a valid IP on the broadcast network.
  3. On the Director PC in vMix, confirm the OMT input source IP/address matches the casting laptop's current IP.
  4. If the IP has changed (e.g., DHCP reassignment), update the OMT source address in vMix accordingly.
  5. If feed is present but degraded, follow network troubleshooting steps above (run cable to switch under OBS 1).
🔌 A device has no power / power strip has a free plug
Hardware ▼

All 8 power strip slots must be filled before booting. A free plug means something is unpowered.

  1. Identify what is missing by checking against the full list: Breakaway Box · Cam 1 · Cam 2 · Cam 3 · GVM Light 1 · GVM Light 2 · HDMI Switcher · Casting Laptop.
  2. Trace the missing device's power cable — it may have been forgotten during setup or come loose.
  3. If the power strip itself appears dead, check that it is firmly plugged into the wall outlet inside the concealed panel and that the panel outlet has power.
  4. Do not boot the laptop until all 8 slots are confirmed filled and all devices have power.
🎛 Stream Deck Mini not showing M / TB buttons in Companion Satellite
Control ▼
  1. Confirm the USB extension cable is firmly seated at both the laptop and the Stream Deck Mini ends.
  2. Check that Bitfocus Companion Satellite is running on the casting laptop — it must be open for the deck to receive its button layout.
  3. If Companion Satellite is open but the deck is blank or showing wrong buttons, try closing and reopening Companion Satellite.
  4. If the deck is still unresponsive, unplug the USB extension from the laptop, wait a few seconds, and replug. Companion Satellite should re-detect the device automatically.
  5. If the issue persists, try plugging the Stream Deck Mini directly into a laptop USB port (bypassing the extension cable) to rule out a faulty extension.
🎵 Music not playing on the MUSIC channel / Foobar2000 not routing
Audio ▼
  1. Check that Foobar2000 is open and playing on the Director PC — it may not have been launched yet or may have stopped.
  2. In Foobar2000, verify the output device is set to route to the RODECaster (check Preferences → Playback → Output).
  3. On the RODECaster, confirm the MUSIC channel fader is up and not muted.
  4. If Foobar2000 is running and routing correctly but the board shows no signal, power cycle the RODECaster and reopen vMix after it restarts.
🎮 Game audio from Observer PCs not heard by casters or on stream
Audio ▼
  1. In OBS on the affected Observer PC, confirm the game capture source has audio enabled and is not muted in the OBS audio mixer.
  2. Verify the Observer PC is transmitting OMT to vMix — if the video feed is also missing, resolve that first (see OMT troubleshooting).
  3. In vMix on the Director PC, check the audio settings for that Observer's OMT input — confirm audio is enabled and its level is not zeroed.
  4. Put that Observer feed on program and check the RODECaster for signal on the game audio channel. If vMix is sending it but the board isn't receiving it, check the USB connection between the Director PC and the RODECaster.
  5. Confirm no solo button is engaged on the board that would mask the game audio channel in your monitor mix.
⏱ Game audio is delayed (often exactly 1 second)
Audio ▼

A consistent 1-second delay is a known OBS sync issue on observer PCs — a full reboot of OBS resolves it reliably.

  1. Identify which observer's feed has the delayed audio.
  2. Fully close OBS on that observer PC and reopen it.
  3. Confirm game capture and audio are transmitting again via OMT before putting that feed back on program.
🔁 Game audio is echoing or doubled
Audio ▼

This happens when both Desktop Audio and Game Capture are passing audio simultaneously in OBS — two slightly offset copies of the same source.

  1. On the affected observer PC, open the OBS audio mixer.
  2. Find the Desktop Audio channel and mute it — only Game Capture audio should be active.
  3. Confirm the echo is gone before returning that feed to program.
